clasp does not generate secrets. You bring your own. Both sides (sender and receiver) must use the same value.

## Generate a secret

```bash
echo "whsec_$(openssl rand -base64 32)"
```

## Use it

Pass the secret directly to `send()` and `verify()`:

```ts
import { send, verify } from "clasp-sh";

await send({
  secret: process.env.WEBHOOK_SECRET!,
  url: "https://example.com/webhook",
  event: { type: "user.created", data: { id: "usr_123" } },
});

const event = await verify({
  secret: process.env.WEBHOOK_SECRET!,
  request,
});
```

## Format

The secret can be any string. clasp handles it like this:

- **Starts with `whsec_`**: the rest is treated as base64-encoded bytes.
- **Anything else**: used as raw UTF-8 bytes.

Both produce a valid HMAC key. Pick one and stay consistent.

## Storage

<Callout type="warn">
  Never commit secrets to git or hardcode them in source files. Use environment
  variables or a secret manager (1Password, AWS Secrets Manager, Doppler, etc.).
</Callout>

Store your secret in an environment variable like `WEBHOOK_SECRET` and rotate it through your secret manager.

## Rotation

To rotate without downtime:

1. Add the new secret alongside the old one on the receiver.
2. Try verifying with the new secret first, then fall back to the old one.
3. Switch the sender to the new secret.
4. Once nothing uses the old secret, remove it from the receiver.

clasp does not have a built-in rotation helper. Handle it explicitly in your code.
