clasp accepts secrets, it doesn't generate them. Use any standard tool.

## Generate one

```bash
openssl rand -base64 32
```

Or with a `whsec_` prefix (Standard Webhooks convention):

```bash
echo "whsec_$(openssl rand -base64 32)"
```

## Use it

```ts
const webhooks = clasp({ secret: process.env.WEBHOOK_SECRET });
```

## Format

The secret can be any string. clasp parses it like this:

- **Starts with `whsec_`** — the rest is treated as base64-encoded bytes.
- **Anything else** — used as raw UTF-8 bytes.

Both formats produce a valid HMAC key. Pick one and stay consistent.

## Sharing

Both sides — sender and receiver — must use the **same** secret value. Store it in an environment variable (`WEBHOOK_SECRET`) and rotate it through your secret manager.

<Callout type="warn">
  Never commit secrets to git or hardcode them in source files. Use environment
  variables or a secret manager (1Password, AWS Secrets Manager, Doppler, etc.).
</Callout>

## Rotation

To rotate without downtime:

1. Add the new secret alongside the old one on the receiver.
2. Try verifying with the new secret first; fall back to the old one.
3. Switch the sender to the new secret.
4. Once you've confirmed nothing uses the old secret, remove it from the receiver.

clasp doesn't have a built-in rotation helper — keep it explicit in your code.
