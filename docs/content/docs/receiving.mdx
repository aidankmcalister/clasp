Use this when you're the **incoming** side — someone else is notifying you. You expose a URL, they POST to it, your service does something with what arrived.

`webhooks.verify()` takes a Web standard `Request` and returns the parsed, verified event. If anything is wrong it throws a [`WebhookError`](/docs/errors).

## Basic usage

```ts
const event = await webhooks.verify(request);
// event.type, event.data — verified
```

## What gets verified

1. The required headers exist (`webhook-id`, `webhook-timestamp`, `webhook-signature`).
2. The timestamp is within tolerance (default 5 minutes).
3. The HMAC signature matches, using a timing-safe comparison.
4. The body is valid JSON with a `type` field.

## Hono

```ts
import { Hono } from "hono";

app.post("/webhook", async (c) => {
  const event = await webhooks.verify(c.req.raw);
  return c.json({ ok: true });
});
```

## Next.js (App Router)

```ts
// app/api/webhook/route.ts
export async function POST(request: Request) {
  const event = await webhooks.verify(request);
  return Response.json({ ok: true });
}
```

## Bun.serve

```ts
Bun.serve({
  async fetch(req) {
    if (req.method === "POST") {
      const event = await webhooks.verify(req);
      return Response.json({ ok: true });
    }
    return new Response("Not Found", { status: 404 });
  },
});
```

## Cloudflare Workers / Deno

Both runtimes use `Request` natively — same call.

```ts
export default {
  async fetch(req: Request) {
    const event = await webhooks.verify(req);
    return Response.json({ ok: true });
  },
};
```

## Express / legacy Node

If your framework doesn't give you a `Request`, pass headers and raw body separately:

```ts
const event = await webhooks.verify({
  headers: req.headers,
  body: rawBodyString,
});
```

<Callout type="warn">
  Always use the **raw** request body for verification. Reserializing parsed
  JSON can change byte order and break the signature.
</Callout>

## Timestamp tolerance

clasp rejects webhooks older than **5 minutes** by default. Override per call:

```ts
await webhooks.verify(request, { tolerance: 60 });  // 1 minute
await webhooks.verify(request, { tolerance: 600 }); // 10 minutes
```

A tighter tolerance hardens against replay attacks but is less forgiving of clock skew. 5 minutes is a sensible default.

## Testing with external services

`localhost` isn't reachable from the internet, so services like Stripe or GitHub can't POST to you directly during development.

For now, expose your local receiver with a tunnel — **ngrok** or **cloudflared**:

```bash
ngrok http 3000
```

Use the public URL it prints (with `/webhook` appended) as your receiver URL in the external service's config.

<Callout>
  A hosted, webhook-specific relay (`bunx clasp-sh tunnel`) is on the roadmap. For now, ngrok is the move.
</Callout>

## Idempotency

Webhooks can arrive multiple times. Use `event.id` (the `webhook-id` header) to deduplicate:

```ts
const event = await webhooks.verify(request);

if (await alreadyProcessed(event.id)) {
  return Response.json({ ok: true });
}

await markProcessed(event.id);
// ... do the work
```
